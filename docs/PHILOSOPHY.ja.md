# 思想（このプロジェクトが何を守るか）

> このドキュメントは実装仕様ではなく、**設計判断のよりどころとなる思想**を記録する。
> 機能を足すか迷ったときは、ここに書かれた原則に反しないかどうかで判断する。

## 中核の約束：デザインデータを外に出さない

このサーバの存在理由は **「デザインデータが私設・ローカル経路の外へ一切出ないこと」** に尽きる。

- S3 にアップロードしない（公式 Figma の画像APIは公開S3 URLを返すが、これを避ける）
- Figma REST / APIトークンを使わない
- 第三者サービスに送らない
- 公開ポートを作らない
- そもそも外向きHTTPを行わない

画像バイトは Figma クライアントの `exportAsync` でローカル生成し、私設の
`gh codespace ports forward` トンネル（GitHub認証付き・公開ポートなし）経由でのみ流れる。

## 今の姿：read-only

現在のツールは9つすべて **読むだけ**。開いているドキュメントから情報を取り出すだけで、
何も書き換えず、何も送らない。これは制約ではなく、上の約束を最も単純な形で満たす設計。

## 拡張するときの大原則

将来 UI 生成などの **書き込み** を足す場合でも、中核の約束を絶対に壊さない。具体的には：

1. **任意コードを実行しない。**
   モデル（LLM）が生成したJSをプラグインで実行しない。任意JSは任意のエンドポイントへ
   `fetch` できてしまうため、「外に出さない」思想と根本的に相容れない。
   → だから公式の `use_figma`（任意JS実行）は**採用しない**。

2. **やり取りするのはコードではなくデータ。**
   モデルが出すのは宣言的な指示（JSON）。プラグインはそれを**固定の操作allowlist**で
   適用するだけ。データは実行経路に乗らない。

3. **ネットワーク面を増やさない。**
   プラグインの通信先は `ws://localhost:3055` のローカルbridgeのみ。新しいドメイン・
   送信系の操作は追加しない。データを外に出す経路を物理的に持たない。

4. **賢さはユーザ自身のIDEエージェントに置く。**
   解釈・計画はユーザが既に信頼している手元のエージェント（Copilot / Claude Code）が担う。
   キャンバスデータを第三者のバックエンドLLMに送らない。
   （＝多くの類似プラグインより、プライバシー軸で強い。）

5. **入力は値として検証する。**
   数値はクランプ、参照は実在するものだけ解決。文字列をデータとして扱い、実行経路に乗せない。

## なぜこの思想か

セキュリティに厳しい組織にとって、デザインが公開URLや第三者に渡ることは許容できない。
「便利さのために外部送出を許す」を一度認めると、この約束は崩れる。だからこのプロジェクトは
**利便性より無送出を優先する**。機能拡張も、この優先順位を変えない範囲でのみ行う。
